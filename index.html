Inception

by: Reuben Beeler

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I have played this game on paper and it was fun. It is also easy to pick up, because it is an expansion of the classic tic-tac-toe, which nearly everyone knows how to play.

2. What I Learned

1) I used outlines for the larger letters to prevent blocking the smaller ones. 2) I highlight the region that is available to play in every turn in the color of the current player. 3) Instructions were not clear originally, later improved to describe every step in detail.

3. Architecture of My Game

The first (main) class called "Inception" starts the game and initializes all values at "", effectively making the board completely empty, how it should appear at the beginning. It also checks for mouse clicks and sends the signal to the object of class GameBoard. The GameBoard class defines the game mechanics, such as the conditions in which a player wins. Finally, the DrawingBoard class uses the default Graphics class from java.awt to draw the game onto the window that is seen by the user(s).

4. Technical Implementation

I like how smoothly the game runs, and is easy to test and look at with unique colors for the two players and the tic-tac-toe boards. I also am proud of the functionality of the colored regions that clearly show where a player can put his/her letter. To achieve this, I broke up the 4 longest lines segments into 3 smaller segments each, and then drew the necessary lines in the color of the current player.

5. Most Challenging Part

The most challenging part of this project was getting started, as it typically is any project. I knew I had to have 3 separate classes, so I created them, determined their uses, and went on from there.

6. Future Fixes and Updates

I would include the options for a third player and single player(PvC). Another addition would be a menu to select the game mode, contain the instructions, and allow playing multiple games without having to restart the program.
